# Claude Code — Navia Project Instructions

## 🗂️ Project Overview
This is **Navia**, a React project built in VSCode. This file contains standing instructions for how Claude Code should behave across every session.

---

## 📓 Obsidian Vault Workflow

A folder called `Navia Vault/` lives inside this project root. It is an Obsidian Vault used as a **persistent memory layer** between Claude sessions. Claude Code is responsible for reading it at the start of each session and writing to it at the end.

### Vault Structure
```
Navia Vault/
├── Sessions/
│   ├── YYYY-MM-DD Session 01.md
│   ├── YYYY-MM-DD Session 02.md
│   └── ...
├── Navia Project Details.md
└── Useful SWE Websites.md
```

---

## ▶️ START OF EVERY SESSION — Do This First

1. **Read the most recent file** in `Navia Vault/Sessions/` (sort by date, take the latest)
2. **Summarize it back** to the user in 3–5 bullet points so they can confirm context is correct
3. **Note any open issues or next steps** flagged in that note
4. Then ask: *"Ready to continue — what are we working on today?"*

If no session notes exist yet, say so and ask the user to describe the current state of the project before starting.

---

## ⏹️ END OF EVERY SESSION — Do This Last

When the user says anything like:
- *"Let's wrap up"*
- *"End of session"*
- *"Write the vault note"*
- *"We're done for today"*

Claude Code should:

1. **Create a new session note** in `Navia Vault/Sessions/` named `YYYY-MM-DD Session NN.md` (auto-increment the session number based on existing files)
2. **Fill it out completely** using the template below
3. **Confirm** to the user: *"Session note written to Navia Vault/Sessions/."*

## 📌 General Behavior Rules

- **Always read the vault first.** Never start coding without checking the latest session note.
- **Never overwrite existing session notes.** Always create a new dated file.
- **Be specific in session notes.** Vague entries like "fixed bug" are not useful. Write "Fixed `useEffect` infinite loop in `MapView.jsx` caused by missing dependency array."
- **Capture the why.** The Key Decisions section is the most valuable part of the note — always fill it out.
- **Git awareness.** When writing the session note, run `git log -1 --pretty=format:"%H %s"` to get the latest commit hash and message and include it in the Current State Summary.
- **Respect the file naming convention.** Session files must follow `YYYY-MM-DD Session NN.md` so they sort correctly in Obsidian.
- **Never read or modify .env files**
- **Never run destructive commands (rm -rf, drop database, etc.)**
- **Never push to git without my explicit approval**
- **Never expose or print API keys or secrets**
