## Key Decisions & Reasoning

| Decision | Reasoning | Alternatives Considered |
|----------|-----------|------------------------|
| Size config object instead of separate props | Single `size` prop is cleaner than separate width/height/baseR props; easy to add new sizes later | Individual numeric props, CSS-only scaling |
| Synthetic days for public constellation page | Public endpoint only returns day_count (no user data leak); constellation generator only needs count + tripId for deterministic layout | Returning full day objects publicly |
| Canvas-based PNG export for sharing | Works across browsers without server-side rendering; resolves CSS variables via static color map | html2canvas library, server-side SVG→PNG |
| Collaborator lookup by email, not username | Email is the unique identifier in the system; simpler UX for inviting | Invite by user ID, invite link system |
| Profile editing inline vs modal | Inline editing is lighter-weight and consistent with the card layout | Full modal form, separate edit page |
