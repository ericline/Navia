## 🧠 Key Decisions & Reasoning

| Decision | Reasoning | Alternatives Considered |
|----------|-----------|------------------------|
| Use SVG `<animate>` for pulse rings instead of CSS `transform: scale()` | CSS transforms on SVG circles scale from SVG origin (0,0), not circle center — causes the ring to drift left/down | `transform-box: fill-box` (inconsistent browser support for SVG) |
| Use `useRef` guard + cleanup `cancelled` flag for auto-generate | React Strict Mode double-fires useEffect, causing `generateDaysForTrip` to run twice and create duplicate days | Debounce (unnecessary complexity), move to server (backend already skips existing dates but race condition between two concurrent calls) |
| Pre-defined constellation templates per star count (2–7) with seeded jitter | Gives each week a recognizable, non-horizontal shape while keeping it deterministic (same seed = same layout on re-render) | Fully random (too chaotic), single template (boring) |
| `PATCH` endpoint with `unschedule` boolean flag | Need to distinguish "set day_id to null" (unschedule) from "don't change day_id" (both would be `null` in JSON) | Separate `/unschedule` endpoint (more routes), use sentinel value like -1 (hacky) |
| Edit/delete icons appear on card hover only | Keeps the UI clean; cards are small so permanent buttons would be cluttered | Context menu (harder to discover), swipe-to-delete (desktop unfriendly) |
